# Guía para elaborar el mapa mental o notas técnicas
# Actividad 10: “Caza errores”

## Objetivo

Representar de manera visual o esquemática los errores frecuentes encontrados en el proyecto web, explicando su diagnóstico, corrección y prevención.

Puedes entregar este punto de dos formas:

1. **Mapa mental**, hecho a mano, en Canva, PowerPoint, Word, XMind, Draw.io, Miro u otra herramienta.
2. **Notas técnicas organizadas**, en un documento escrito con títulos, tablas o esquemas.

---

# Opción A. Mapa mental

## Tema central

En el centro del mapa escribe:

**Errores frecuentes en HTML y CSS: diagnóstico, corrección y documentación**

---

## Ramas obligatorias del mapa mental

Tu mapa debe tener como mínimo estas seis ramas principales:

### 1. Errores en HTML

Incluye ejemplos como:

- Rutas incorrectas.
- Etiquetas mal escritas.
- Identificadores incorrectos.
- Clases mal digitadas.
- Enlaces internos que no coinciden con las secciones.
- Llamadas incorrectas a funciones de JavaScript.

### 2. Errores en CSS

Incluye ejemplos como:

- Archivo CSS mal enlazado.
- Selectores que no coinciden con el HTML.
- Variables CSS inexistentes.
- Propiedades inválidas.
- Media queries mal escritas.
- Problemas de responsive design.

### 3. Errores en JavaScript

Incluye ejemplos como:

- Archivo JS mal enlazado.
- Errores de sintaxis.
- Funciones llamadas con nombre incorrecto.
- Identificadores del DOM que no coinciden.
- Validaciones que no funcionan.

### 4. Herramientas de diagnóstico

Incluye:

- Inspeccionar elemento.
- Consola del navegador.
- Pestaña Network/Red.
- Revisión manual del código.
- Validadores HTML y CSS.
- Prueba en diferentes tamaños de pantalla.

### 5. Proceso de corrección

Organiza el proceso así:

1. Observar el problema.
2. Identificar el archivo afectado.
3. Diagnosticar la causa.
4. Corregir el código.
5. Guardar los cambios.
6. Probar nuevamente.
7. Documentar la solución.

### 6. Buenas prácticas

Incluye recomendaciones como:

- Usar nombres claros de archivos y carpetas.
- Revisar mayúsculas, minúsculas y extensiones.
- Mantener HTML, CSS y JS organizados.
- Comentar el código cuando sea necesario.
- Probar cambios uno por uno.
- Documentar errores y soluciones.

# Opción B. Notas técnicas

Si no haces mapa mental, puedes entregar notas técnicas con esta estructura:

## 1. Errores encontrados

Escribe una lista de los errores que encontraste en el proyecto.

## 2. Clasificación de errores

Clasifica los errores en una tabla:

| Tipo de error | Ejemplo encontrado | Archivo afectado |
|---|---|---|
| HTML | Ruta incorrecta del logo | index.html |
| CSS | Propiedad inválida | css/styles.css |
| JavaScript | Error de sintaxis | js/app.js |

## 3. Herramientas usadas

Explica qué herramientas usaste para diagnosticar los errores.

## 4. Correcciones aplicadas

Resume las correcciones más importantes.

## 5. Aprendizaje final

Escribe una reflexión de mínimo 8 líneas sobre lo que aprendiste.

---

# Criterios mínimos de entrega

El mapa mental o las notas deben cumplir con lo siguiente:

- Deben mencionar errores de HTML y CSS.
- Deben explicar cómo se diagnosticaron los errores.
- Deben incluir al menos 5 errores concretos del proyecto.
- Deben mostrar la relación entre problema, causa y solución.
- Deben tener orden visual o estructura clara.
- Deben incluir mínimo 5 buenas prácticas para evitar errores futuros.

---

# Formato de entrega sugerido

Puedes entregar el mapa mental o las notas en cualquiera de estos formatos:

- PDF.
- Imagen PNG o JPG.
- Documento Word.
- Presentación PowerPoint.
- Archivo Markdown.

Nombre sugerido del archivo:

```text
mapa_mental_caza_errores_nombre_apellido.pdf
```
